# 手工烤面筋微信小程序需求文档（PRD）

## 1. 项目背景

随着夜市、小吃摊、外卖及私域流量经营的发展，手工烤面筋商家需要一个轻量、低成本、易推广的线上工具，用于展示商品、接单、管理订单及维护老客户。微信小程序具备天然流量入口和便捷支付能力，适合作为核心销售与运营平台。

## 2. 项目目标

* 为顾客提供便捷下单体验
* 为商家提供高效订单与商品管理能力
* 支持线上线下一体化经营
* 建立用户复购与私域沉淀能力

## 3. 目标用户

### 3.1 C端用户

* 夜市/小吃摊消费者
* 外卖用户
* 社区团购用户

### 3.2 B端用户

* 手工烤面筋个体商户
* 小吃连锁品牌

## 4. 使用场景

1. 顾客扫码摊位二维码进入小程序下单
2. 顾客线上预订，到店自取
3. 顾客外卖配送下单
4. 老客户在小程序复购

## 5. 功能架构

* 门店信息展示

* 加料选择
* 数量选择

#### 5.1.4 购物车

* 删除商品

#### 5.1.5 下单与支付

* 选择配送方式（自取/配送）

## 6. 核心业务流程

### 6.1 下单流程

用户进入小程序 → 选择商品 → 加入购物车 → 提交订单 → 微信支付 → 商家接单 → 制作 → 完成

### 6.2 商家处理流程

接收订单 → 确认制作 → 完成订单 → 数据统计

## 8. 技术需求

* 微信小程序原生框架
* 后端：Node.js / PHP / Java 任一
* 数据库：MySQL / MongoDB
* 支付：微信支付
* 消息：订阅消息推送

---

## 9. 性能要求

* 页面加载时间 ≤ 2秒
* 支付成功率 ≥ 99.9%
* 并发支持 ≥ 1000用户
